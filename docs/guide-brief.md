# 依頼：ゴルフコンペ予約チェック管理ツール「スタッフ用 使い方ガイド」LP作成

## 依頼概要
社内ツール「予約チェック管理」のスタッフ向け使い方ガイドのランディングページ（1ページ）を作ってください。日本語・ゴルフ場の受付/予約担当者向け・専門用語なしの平易な説明で。GitHub Pages配信を想定した静的サイト（単一HTML推奨、スマホ閲覧にも対応）。

## 対象アプリ
- 名称：予約チェック管理（ゴルフコンペ予約のフォロー連絡を管理するツール）
- リポジトリ：https://github.com/nkhtnkht-ops/compe-demo （`index.html` が本体）
- 公開アプリ：https://nkhtnkht-ops.github.io/compe-demo/
- ※ デザインと正確な挙動は `index.html` を参照して合わせること。

## デザイン要件（アプリと統一：Figma DESIGN.md ベースのモノクロ基調）
- インターフェースは**黒(#000)と白(#fff)のモノクロ**。色は「コンテンツ（状態表示）」にだけ使う。
- ボタン/タブは**ピル形状（border-radius:50px）**、アイコンボタンは円(50%)。
- ラベルや小見出しは**等幅フォント（SF Mono系）＋大文字風＋字間広め**の技術ラベル調。
- カードは**白背景＋ヘアライン枠＋角丸8px**、影は控えめ。フォーカスは**破線2px**。
- フォントは軽めのウェイト中心、本文は負の字間。フォント指定：`"SF Pro Display", system-ui` / 等幅は `"SF Mono", Menlo`。
- ステータス色（コンテンツ）：要対応=橙系、遅延/繰越=赤系、完了=緑系。

## 掲載する使い方コンテンツ（このとおりに正確に）

### これは何
- ゴルフコンペ予約に対し、予約後の決まったタイミングで代表者へ連絡（コンタクト）する作業を「やり漏れなく」管理するツール。
- 予約1件ごとに、4回のコンタクト①〜④の対応状況を記録していく。

### コンタクト①〜④（連絡タイミング）
- ①受付翌日／②60日前／③35日前／④18日前（日数は設定で事業所ごとに変更可）。
- 各コンタクトの状態：**〇＝連絡済／不在＝つながらず繰越／不要／空＝未対応**。
- ほかに **組数確定（〇 or キャンセル）** と **組合せ入力（済/未）** を記録。組合せ入力が済むと追跡終了。

### 画面の基本
- 上部の **「今日やること／全件」** 切替：今日やること＝対応が必要な予約だけ、全件＝すべて。
- 左＝予約一覧（カード）、右＝詳細パネル。カードは「氏名／日付・曜日／コース・時間・組数／状態バッジ」。
- 並び替え：**プレー日順／受付日順**。

### 主な操作
- **状態を付ける**：カードや詳細で 〇／不在／不要 を選ぶ（即消えず、更新するまで一覧に残る）。
- **氏名検索**：上部の検索ボックスで氏名から該当予約へ。
- **コンタクトフィルタ**：「全部／①／②／③／④」。選ぶと、そのコンタクトが“期限到来かつ未対応/不在”の予約だけを全件から抽出＝その回の架電キュー（①を続けて処理する等に便利）。
- **期限の見え方**：遅延●日（赤）／本日／あと●日（予告）／繰越（不在）。
- **不在時の次回連絡日**：不在の予約に再架電日を設定すると、その日まで「今日やること」から外れ、当日に再表示。
- **④までスキップ**：メンバー・定例コンペ向け。①〜③を一括「不要」にして④だけ残す。
- **元に戻す**：直前の状態変更を1つずつ取消（「更新（整理）」を押すまで有効。削除・取込は対象外）。
- **⟳ 更新（整理）**：解決済みを「今日やること」から外して一覧を整理。
- **Excel書き出し**：現在の台帳を.xlsxでバックアップ（再取込可能。組合せ入力も保持）。

### 設定
- **初回起動時メニュー**（初回セットアップ用・通常運用では使わない）
  - ① 新規台帳作成：空のJSON台帳ファイルを新規作成。
  - ② 既存台帳移行取り込み：現行のExcel台帳(.xlsm/.xlsx)を全件取り込み（既存データは置換）。
- **コンタクトのタイミング（日数）**：①〜④の日数を事業所ごとに変更。
- **表示オプション**：
  - 「今日やること」に表示するコンタクト①〜④の ON/OFF（**②は既定オフ**。事業所により②も使用可）。
  - 組数確定したら「今日やること」から外す ON/OFF。

### ファイルと保存（重要：運用の肝）
- データは共有ドライブ上のJSON台帳に保存（File System Access API）。**編集は自動保存**。
- 初回：設定の「初回起動時メニュー」で新規作成 or 既存台帳移行。既に台帳がある人は上部 **「開く」** で接続。
- 一度つなげば「どのファイルか」はブラウザが記憶（毎回選び直し不要）。
- ブラウザを再起動すると権限が切れることがあり、その時だけ **「権限再付与」を1回クリック**（ファイルは記憶済みなので選び直し不要）。
- ※ Chrome/Edge推奨（File System Access API対応ブラウザ）。

8. フッター（リポジトリ/アプリへのリンク）

## トーン・その他
- 日本語、スタッフ向けの平易な言い回し。専門用語は噛み砕く。
- アプリと同じモノクロFigma基調で、見た目に一貫性を持たせる。
- 各操作は「いつ使うか→どうするか」の順で。スクリーンショット用の差し込み枠（プレースホルダ）があると尚良い。
