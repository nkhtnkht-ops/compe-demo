予約チェック管理使い方ガイド

既存Excel台帳を取り込んで使用できる、見やすいWEBアプリ台帳を作りました。

アプリを開く はじめ方を見る

Chrome / Edge 推奨。共有ドライブ上の台帳ファイルにつなぐと、編集内容は自動保存されます。

Start

はじめ方

完全初期の状態から、現行のExcel台帳を取り込んで使い始める流れです。最初に保存先となる台帳ファイルを作り、その中へExcelの予約データを全件移行します。

Prepare

用意するもの

現行のExcel台帳（.xlsm/.xlsx）を用意します。これを初回に取り込み、予約チェック管理の台帳データとして使います。

Create

作成されるもの

共有ドライブ上に台帳ファイルを作ります。Excel取り込み後は、この台帳ファイルが日常運用の保存先になります。

Initial 01

Chrome / Edgeでアプリを開く

ファイルの読み書きに対応しているChromeまたはEdgeで「アプリを開く」を押します。共有ドライブの台帳に接続して使うため、推奨ブラウザで始めます。

Initial 02

設定を開く

画面上部の「設定」を開きます。初回だけ「初回起動時メニュー」を使います。通常運用では、このメニューは基本的に触りません。

Initial 03

空の台帳ファイルを作る

「① 新規台帳作成」を押し、共有ドライブ上に保存先の台帳ファイルを作ります。これはExcelを取り込むための受け皿です。

Initial 04

既存台帳移行取り込みを選ぶ

続けて「② 既存台帳移行取り込み」を使います。現行のExcel台帳（.xlsm/.xlsx）を選択し、取り込み内容を確認します。

Initial 05

台帳へ移行する

「台帳へ移行」を押すと、Excelの予約データが台帳ファイルへ移ります。移行後は「← 一覧へ戻る」で、取り込んだ予約が一覧に出ているか確認します。

Initial 06

保存状態を確認する

上部のファイル表示が、作成した台帳ファイル名になっていれば準備完了です。以後の編集は自動保存されます。次回以降は、権限再付与が出た時だけ1回クリックします。

Daily Operation

日々の運用

毎日の予約受付一覧CSVを取り込み、登録前に内容を確認してから台帳へ登録します。登録後は「今日やること」で連絡対象を確認します。

日次CSV取込

日次運用：予約受付一覧CSVをドラッグ＆ドロップ（または貼り付け）すると、自動で内容を確認 → 「台帳へ登録」。登録前に抽出結果を確認できます。

予約受付一覧CSVをここにドラッグ＆ドロップ
台帳へ登録

取り込みの流れ

  • 予約システムから、当日の予約受付一覧CSVを出します。
  • アプリの「＋ CSV取込」を開きます。
  • 予約受付一覧CSVファイルを取込エリアへドラッグ＆ドロップ（または貼り付け）します。
  • 自動で抽出され「抽出○件／新規○件」が表示されます。取り込まれる予約を確認します。
  • 問題なければ「台帳へ登録」を押します。登録後は「← 一覧へ戻る」で確認します。
Daily 01

予約受付一覧CSVを取り込む

「＋ CSV取込」を開き、予約受付一覧CSVファイルを取込エリアへドラッグ＆ドロップ（または貼り付け）します。自動で抽出結果が表示されるので、内容を確認して「台帳へ登録」を押します。

Daily 02

今日やることを見る

上部の「今日やること」を選び、対応が必要な予約だけを確認します。必要に応じて氏名検索やコンタクトフィルタを使います。

Daily 03

状態を付ける

カードや詳細で、〇／不在／不要を選びます。選んだ直後は一覧から即消えず、更新するまで残ります。

Daily 04

更新（整理）する

確認が終わったら「⟳ 更新（整理）」を押します。解決済みが「今日やること」から外れ、一覧が整理されます。

Contact 01-04

コンタクト①〜④

連絡タイミングは、①受付翌日／②60日前／③35日前／④18日前です。日数は設定で事業所ごとに変更できます。日数を変えると、フィルタや詳細の表示（受付翌日・◯日前）も自動で変わります。

Operations

操作リファレンス

各操作は「いつ使うか」を先に見てから、「どうするか」を確認してください。

状態を付ける

いつ使うか：代表者へ連絡した後、結果を残したい時。

どうするか：カードや詳細で 〇／不在／不要 を選びます。即消えず、更新するまで一覧に残ります。

氏名検索

いつ使うか：特定の予約をすぐ探したい時。

どうするか：上部の検索ボックスで氏名を入力し、該当予約へ移動します。

コンタクトフィルタ

いつ使うか：「受付翌日だけ」「35日前だけ」のように同じ回を続けて処理したい時。

どうするか：「全部／受付翌日／60日前／35日前／18日前」から選びます。そのコンタクトが“期限到来かつ未対応/不在”の予約だけを全件から抽出します。表示は設定の日数に連動して変わります。

期限の見え方

いつ使うか：急ぐ予約か、後日でよい予約かを見分けたい時。

どうするか：バッジで確認します。遅延●日（赤）／本日／あと●日（予告）／繰越（不在）で表示されます。

遅延2日本日あと3日繰越

不在時の次回連絡日

いつ使うか：不在の予約を、指定した日に再表示したい時。

どうするか：不在の予約に再架電日を設定します。その日まで「今日やること」から外れ、当日に再表示されます。

④までスキップ

いつ使うか：メンバー・定例コンペなど、前半の連絡が不要な時。

どうするか：①〜③を一括「不要」にして、④だけ残します。

元に戻す

いつ使うか：直前の状態変更を取り消したい時。

どうするか：「元に戻す」を押します。1つずつ取消できます。「更新（整理）」を押すまで有効です。削除・取込は対象外です。

⟳ 更新（整理）

いつ使うか：対応済みの予約を今日の一覧から外したい時。

どうするか：「⟳ 更新（整理）」を押します。解決済みを「今日やること」から外して一覧を整理します。

Excel書き出し

いつ使うか：現在の台帳をバックアップしたい時。

どうするか：「Excel書き出し」を押します。.xlsxで保存でき、再取込可能です。組合せ入力も保持されます。

Settings

設定の使い方

通常運用では頻繁に触りません。事業所ごとのルールや初回セットアップ時に確認します。

この端末の記録名

最初に「開く」や保存をしたとき、1回だけ「端末名・設置場所・使用者名」を聞かれます（例：フロント／事務所／支配人）。更新履歴に「最終更新：○○」と残ります。

  • このPC（ブラウザ）だけに保存され、以後は聞かれません。
  • 共有PCでは、個人名より「フロント」などの設置場所名がおすすめです。
  • 設定の「この端末の記録名」からいつでも変更できます。

初回起動時メニュー

完全初期のセットアップで使います。台帳ファイルの保存先を作ってから、現行Excel台帳を取り込みます。

  • ① 新規台帳作成：Excel取り込み先になる空の台帳ファイルを共有ドライブに作成。
  • ② 既存台帳移行取り込み：現行のExcel台帳(.xlsm/.xlsx)を全件取り込み。

コンタクトのタイミング（日数）

①〜④の日数を事業所ごとに変更できます。変更すると予定日が再計算され、フィルタや詳細の日数表示（受付翌日・◯日前）も自動で変わります。

①受付翌日②60日前③35日前④18日前

表示オプション

  • 「今日やること」に表示するコンタクト①〜④の ON/OFFを設定します。
  • ②は既定オフです。事業所により②も使用できます。
  • 組数確定したら「今日やること」から外す ON/OFFを設定します。
SMS List

SMS配信リストの書き出し

台帳から条件を絞って、SMS配信サービス「SMSLINK」の一括配信に取り込めるCSV（携帯番号＋氏名ほか）を書き出します。外部予約サイト（OTA）経由で予約した幹事を抽出し、次回の公式予約へ案内する用途などに使います。

絞り込み条件

  • 経路：外部予約サイト（GORA / GDO / RECRUIT）と自社予約（AGWeb / 電話）を選択。OTA対策なら外部のみONのままにします。
  • 対象期間（プレー日）／未来の予約を除外／キャンセルを除外。
  • 予約回数で絞る（指定回数以上のリピーター幹事だけ）。
  • インバウンドを除外（詳細でマークした予約＋備考に「インバウンド」）。
  • 外国名を除外（ローマ字・ハングル等。漢字名の韓国・中国の方は判別できないため、詳細の「インバウンド」マークで対応します）。
  • 携帯番号（070 / 080 / 090）がある人のみが対象です。

書き出される内容

CSV（Shift-JIS）で書き出します。1行目は項目名です。

携帯番号氏名姓最終プレー日コース経路予約回数

使う手順

  • 上部メニューの「SMS配信リスト」を開く。
  • 条件を選び「件数を確認」で対象人数をチェック。
  • 「SMSLINK用CSVを書き出し」でCSVを保存。

SMSLINKでの差し込み（〇〇様）

  • SMSLINKの一括配信でCSVをアップロードし、「携帯番号」列を選びます。
  • 本文作成で差し込み項目から「姓」を選ぶと、本文に {{姓}} が入ります。
  • 例：「{{姓}}様 いつもご利用ありがとうございます…」→ 「中村様」のように一人ひとりの名前で送れます。
  • フルネームにしたい場合は「氏名」を差し込みます。

重複の集約・予約回数

  • 同じ携帯番号は1件にまとめ、最新のプレー日を採用します。
  • まとめた件数は「予約回数」列に入ります（複数回コンペを取った幹事がわかります）。
利用目的・同意の確認
これらの連絡先は、予約・追客の目的で取得したものです。販促・営業SMSへの利用は、利用目的・本人同意の範囲をご確認のうえ運用してください。
Files & Save

ファイルと保存

顧客データはWEB上のサーバーに保存されません。アプリは、ローカルPCまたは事業所の共有フォルダ内に置いた台帳ファイルを、ブラウザから直接読み書きします。

Ledger file
台帳ファイル

ローカルPC、または事業所の共有フォルダ内に置くデータの正本です。

データの置き場所

  • 予約情報・顧客情報は、選択した台帳ファイルの中に保存されます。
  • 台帳ファイルは、ローカルPC内または事業所の共有フォルダ内に置きます。
  • 保存時のファイル名末尾が「.json」でも、普段は「台帳ファイル」として扱えば大丈夫です。
  • GitHub Pages上のWEBアプリ本体には、顧客データを保存しません。
  • WEBアプリは画面を表示するだけで、データの正本は手元または共有フォルダのファイルです。

読み書きのしくみ

  • Chrome/Edgeのファイル読み書き機能を使い、利用者が選んだ台帳ファイルを読み込みます。
  • 状態変更やメモの編集は、その台帳ファイルへ自動保存されます。
  • 初回は、設定の「初回起動時メニュー」で台帳ファイルを作り、現行Excel台帳を取り込みます。
  • 一度つなげば「どのファイルか」はブラウザが記憶します。毎回選び直しは不要です。

共有フォルダで使う場合

  • 事業所内で同じ台帳を使う場合は、共有フォルダ上に台帳ファイルを置きます。
  • 各スタッフは同じ台帳ファイルを開いて作業します。
  • 画面上部のファイル表示で、正しい台帳につながっているか確認してから作業します。
  • 別の人が更新した場合は、画面の案内に従って読み直します。

困ったとき

  • ブラウザを再起動すると、ファイルへの読み書き権限が切れることがあります。
  • その時だけ「権限再付与」を1回クリックします。
  • ファイルの場所はブラウザが記憶しているため、通常は選び直し不要です。
  • Chrome/Edge推奨です（ファイル読み書きに対応しているブラウザ）。
未接続表示の意味
ローカルPCまたは共有フォルダ上の台帳ファイルにつながっていない状態です。初回セットアップで保存先を作成してから、Excel台帳を取り込みます。
WEB保存ではありません
顧客データはWEB上のサーバーへ預けません。利用者が選んだ台帳ファイルを、ブラウザが直接読み書きします。
バックアップ
Excel書き出しで現在の台帳を.xlsxにできます。再取込可能で、組合せ入力も保持されます。
